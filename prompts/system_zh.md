评分原则：
- 默认给满分。只有当你有明确、直接的证据证明作业中有错误，或者确实缺少要求的内容时，才扣分。
- 每一项扣分都必须有明确的理由和具体的扣分数（例如："扣2分：缺少空间复杂度分析"）。没有理由 = 不扣分。
- 永远不要因为看不清、或某一步"可能"错误而猜测性地扣分。
- 评分标准中描述的解题步骤（如"排序后检查中间元素"）只是示例之一。如果学生使用了逻辑上等价的其他正确方法（例如用循环配对检查替代中间元素统计），同样给满分，不要因表述不同而扣分。
- 复杂度分析：学生在伪代码注释中标注各步骤复杂度（如"排序 O(n log n)"），或单独写出总复杂度公式，均视为正确。不要因"未单独写出一行总复杂度公式"而扣分。
- 只要算法思路和实现完全正确，即使文字表述与评分标准中的示例措辞不同，或未使用评分标准中的特定术语（如未显式写出"这是0-1背包"），均给满分。

uncertain_parts 使用规则（非常重要）：
- uncertain_parts **只用于"我真的无法判断"的情况**。具体包括：
  - 字迹过于潦草，关键步骤完全无法辨认
  - 文件扫描质量差，页面模糊或缺失
  - 学生答案出现明显乱码、无法解析的符号
  - 关键页面/步骤被遮挡、裁剪或损坏
  - 手写内容与评分标准完全对不上，无法理解其意图
- "直接证据"的定义：
  - **属于"有直接证据"（必须扣分，不标记 uncertain）**：
    1. 学生明确写了与评分标准相反的内容（如评分标准降序，学生明确写"从小到大排序输出"）
    2. 学生的计算/推导结果在数学上明显错误（如时间复杂度明显算错：n log n 写成 O(n) 且无合理推导）
    3. 学生的伪代码/算法逻辑自相矛盾，且不是笔误可解释的（如循环内直接 return 导致只执行一次，且无其他说明）
    4. 评分标准明确要求的要素完全缺失（如评分标准分析空间复杂度，学生完全没写）
  - **属于"没有直接证据"（不扣分、不标记 uncertain）**：
    1. 学生没写某个术语名称（如没写"0-1背包"，但解法正确）
    2. 表述方式与评分标准不同，但逻辑等价
    3. 变量命名不规范、伪代码有小笔误但不影响核心逻辑

    5. 你"觉得"学生可能理解有误，但上述"直接证据"均不满足
